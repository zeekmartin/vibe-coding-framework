# Contributing

Thanks for your interest!

## Principles
- Keep the framework lightweight and practical.
- Prefer clarity over complexity.
- Avoid adding dependencies unless strictly necessary.

## Suggested contributions
- Improvements to prompts
- Better checklists
- Examples of adoption in real projects
- CI snippets that remain optional

## How to contribute
1. Fork the repo
2. Create a feature branch
3. Open a PR with a clear description and rationale

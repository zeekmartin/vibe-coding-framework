# Graphic Charter (UI/UX)

Keep the product visually consistent even when iterating quickly with AI.

## Principles
- Clarity over cleverness
- Consistency over novelty
- Accessibility by default

## Typography
- Define your font(s), sizes, weights
- Define heading scale (H1/H2/H3)

## Spacing & layout
- Use a spacing scale (e.g., 4/8/12/16/24/32)
- Prefer grids over free-form placement

## Components
List reusable UI components and rules:
- Buttons (variants, sizes)
- Inputs (states)
- Cards (padding, radius)
- Alerts (severity)

## Copywriting
- Tone of voice
- Error message style
- Microcopy conventions

## Do/Don’t
- Do: reuse components, match spacing scale
- Don’t: introduce new styles without updating this document
